Polymorphism

-- run time => Method overriding
-- compile time => Method overloading

Final keyword

-- Variable => The value/reference cannot be changed after assignment.
-- Method => The method cannot be overridden by subclasses
-- Class => The class cannot be inherited or extended.

Annotation

@Override => Used when a method overrides a parent class method.
@Retention =>	Specifies how long the annotation is available
@Target => Specifies where the annotation can be used
@Inherited =>	Allows child classes to inherit annotation
@Documented	=> Includes annotation in Javadoc
@Deprecated => Marks a method or class as outdated.

Types of Interface

-- Noramal interface =>  two or more abstract method
-- Functional Inteface => only one abstract method
-- Marker Interface => no abstract method

Errors 

-- Runtime Errors => These occur while the program is running. They are usually caused by exceptions.
                  => example divide by zero

-- Compile time Errors => (Syntax Errors) These errors are detected by the compiler before the program runs.
                       => example Missing semicolon
                       
-- Logical Errors => The program compiles and runs, but produces the wrong output.
                  => example incorrect formula
